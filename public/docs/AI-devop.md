作为前端开发，要让AI精准复现UI设计并匹配业务需求，需要构建「需求+视觉+技术」三位一体的提示语框架，核心是让AI同时理解「业务逻辑」「视觉细节」和「前端实现规则」。以下是经过实战验证的模板和操作流程：

### 一、基础提示语模板（核心框架）

```
角色：你是有5年前端开发经验的工程师，精通${技术栈}和${UI框架}，能精准将设计图转化为像素级还原的代码，熟悉${交互规范}（如hover效果、加载状态）。

业务需求核心：
1. 页面用途：${如「电商商品详情页，用于展示商品信息和下单」}
2. 核心功能：${从需求文档提炼，如「商品图片轮播、规格选择（颜色/尺寸）、加入购物车按钮、库存实时显示」}
3. 数据来源：${如「静态模拟数据」「后端接口字段（附字段说明）」}

UI设计参考：
- 设计图地址：${你的UI图链接}
- 关键视觉要求：${从设计图提取的细节，如「主色调#FF4400，按钮圆角8px，标题字体为PingFang SC 20px bold，列表项间距16px」}
- 响应式要求：${如「适配375px手机/1200px桌面，移动端导航转为底部tab」}

技术约束：
- 技术栈：${如「HTML5 + Tailwind CSS v3 + JavaScript原生」「React 18 + TypeScript + Ant Design」}
- 交互细节：${如「点击规格选项时添加边框高亮，库存不足时按钮置灰并提示」}
- 性能要求：${如「图片懒加载，列表项复用减少DOM节点」}

输出格式：
1. 实现思路：说明布局结构（如「采用flex布局，分为header/body/footer，body内左侧图片区+右侧信息区」）
2. 完整代码：按「HTML结构→CSS样式→JavaScript逻辑」拆分，带注释标注与设计图对应的细节（如「此处对应设计图中红色按钮，圆角8px」）
3. 适配说明：列出不同屏幕尺寸的适配方案，标注设计图中未明确的默认处理（如「移动端图片占满宽度，隐藏次要信息」）
4. 待确认点：指出设计图中可能存在歧义的地方（如「规格选择区未标注选中状态颜色，暂用主色调#FF4400实现」）
```

### 二、关键信息补充技巧（提升精准度）

#### 1. 给UI设计图「打标签」（解决AI看不到图的问题）

AI无法直接访问设计图链接，需要你手动提取核心视觉参数，按「区域→元素→样式」结构化描述：

```
UI设计细节补充：
- 顶部导航栏：高度60px，背景色#FFFFFF，阴影0 2px 8px rgba(0,0,0,0.1)，左侧logo尺寸40x40px，右侧菜单文字14px，间距24px
- 商品图片区：宽400px（移动端100%），高500px，边框1px solid #EEEEEE，圆角12px，底部缩略图栏高度80px，间距8px
- 加入购物车按钮：宽100%，高50px，背景色#FF4400，文字白色16px，hover时背景色#E53E00，点击有0.2s缩放动画（0.98倍）
```

#### 2. 业务逻辑「拆步骤」（避免功能遗漏）

从需求文档中提炼「用户操作→系统反馈」的流程，让AI理解交互背后的逻辑：

```
业务交互流程：
1. 规格选择：用户点击颜色/尺寸选项→选中项添加高亮边框→实时计算选中规格的库存（若库存=0，按钮置灰并显示「无货」）
2. 数量调整：点击「+」按钮→数量+1（最大不超过库存）→小计金额实时更新（单价×数量）
3. 加入购物车：点击按钮→显示「加入成功」toast（3秒后消失）→按钮暂时变为「已加入」（1秒后恢复）
```

#### 3. 技术实现「定标准」（统一代码风格）

指定具体技术规范，减少后期调整成本：

```
代码规范：
- CSS：使用Tailwind工具类优先，避免自定义CSS；颜色用变量（如--primary: #FF4400）；响应式用sm: md: lg:前缀
- JavaScript：事件处理用事件委托；数据处理抽离为单独函数（如calcTotalPrice()）；DOM操作使用querySelector
- 组件化：若用React，拆分为<ProductImage /> <SpecSelector /> <AddToCartButton />等组件，使用props传递数据
```

### 三、迭代优化策略（快速修稿）

首次生成代码后，若存在偏差，用「对比式提示」让AI精准修正：

```
当前问题：
1. 视觉偏差：按钮圆角实际为4px，设计图要求8px；标题字体实际为18px，设计图要求20px
2. 功能缺失：未实现「规格未选时点击加入购物车，提示『请选择规格』」的逻辑
3. 响应式问题：移动端（375px）时商品图片右侧出现10px空白，应占满屏幕宽度

修正要求：
- 按设计图调整按钮圆角为8px，标题字体改为20px（font-weight: 600）
- 补充规格未选时的提示逻辑（用alert或toast组件）
- 移动端图片区添加width: 100% !important，移除右侧margin

请只输出修改后的代码片段（标注修改位置），无需重复完整代码
```

### 四、实战案例（电商商品详情页）

```
角色：你是有5年前端开发经验的工程师，精通HTML5 + Tailwind CSS v3 + JavaScript，能精准还原设计图，熟悉电商交互规范。

业务需求核心：
1. 页面用途：服装类商品详情页
2. 核心功能：商品轮播图（主图+5张缩略图）、规格选择（颜色3种/尺寸4种）、数量调整（1-库存）、加入购物车、立即购买
3. 数据来源：静态数据（商品名：「夏季纯棉T恤」，单价：99元，库存：各规格不同）

UI设计参考：
- 设计图地址：https://xxx.com/design.png
- 关键视觉要求：主色调#2D7FF9，按钮圆角8px，标题用18px PingFang SC bold，价格用24px red bold，规格选中项有2px蓝色边框
- 响应式要求：移动端（≤768px）时，轮播图占满屏幕宽度，规格选项一行2个，按钮占满宽度

技术约束：
- 轮播图实现：用原生JS实现（自动播放+手动切换+缩略图联动）
- 交互细节：规格选择后实时显示对应库存；数量不能超过库存；加入购物车后显示成功提示（绿色背景，2秒消失）

输出格式：...
```

### 核心原则：让AI「有参照物可依」

1. **视觉参数越具体越好**：不要说「按钮要大一点」，而说「按钮宽度300px，高度50px，字体16px」（可从设计工具中直接获取数值）。
2. **业务逻辑「可视化」**：用「用户做什么→页面发生什么」的句式，避免抽象描述（如不说「处理规格逻辑」，而说「用户选红色+M码时，库存显示20件」）。
3. **分阶段生成**：复杂页面先让AI实现静态布局，确认视觉无误后，再让其补充交互逻辑，减少一次性生成的偏差。

按这个框架操作，能将AI生成代码的还原度提升至80%以上，剩下20%只需微调即可，比纯手写效率提升3-5倍。如果需要针对特定UI框架（如Vue/React）的细分模板，可以告诉我，我会补充更具体的细节。
